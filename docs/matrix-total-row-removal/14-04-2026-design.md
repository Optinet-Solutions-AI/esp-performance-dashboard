# Design: Remove Redundant ESP Total Rows from Matrix View

**Date:** 14-04-2026
**Author:** Leo Sulano
**Status:** Implemented

---

## Problem

The ESP Deliverability Matrix table displayed a `{espName} — Total` row directly below every ESP header row. This row showed the exact same aggregated metrics already visible in the ESP header row itself, resulting in redundant, visually noisy data.

Affected rows:
- **Collapsed state:** ESP header row → `{espName} — Total` row (identical data)
- **Expanded state:** ESP header row → IPs → From-domains → `{espName} — Total` grand total (same as header)

---

## Decision

Remove both ESP-level total rows. The ESP header row is sufficient — it already displays the aggregated totals for that ESP.

IP-level and from-domain totals are retained as they provide additional context when drilling into the hierarchy.

---

## Implementation

**File:** `src/components/views/MatrixView.tsx`

### Change 1 — Collapsed-state total row

**Before:**
```tsx
if (!espEx) {
  rows.push(
    <tr key={espKey + '-total'}>
      <td ...>{espName} — Total</td>
      <td ...></td>
      <DataRow agg={espTot} isTotal />
    </tr>
  )
  return
}
```

**After:**
```tsx
if (!espEx) return
```

### Change 2 — Expanded-state grand total row

**Before:**
```tsx
// ESP grand total
rows.push(
  <tr key={espKey + '-grand-total'}>
    <td ...>{espName} — Total</td>
    <td ...></td>
    <DataRow agg={espTot} isTotal />
  </tr>
)
```

**After:** Block removed entirely.
